    Overview

    Background¶

    この研究室におけるBackground¶

    サーボモーターを連結させて作った魚ロボットが，LightHillの曲線に乗るように制御する（プログラムする）ことに力を入れており，すでに多くのケースで成功している．これをさらに発展させることに加えて，これを利用して，魚の効率的な泳ぎ方についてもより深く調べていきたい．ただ，その効率を計測する方法が難しい．

    実際の魚に近づける

    • 目的の運動をするための機構やプログラムを作成する
    • 実際の魚の形にも近づける
    • 外部につながる線もなくしたい

    計測方法の改善

    • モーションキャプチャーを使って，魚の動きを計測する

    この研究分野におけるBackground¶

    水中ロボットに関連する背景は，日本語で書かれた藤原(2016)の第１章序論を読んでみてください．

    Proposing Research Topics¶

    効率的な水中ロボットを開発するためのアプローチは以下の二つに分けられるだろう．

    • 理論的なアプローチ
    • 探索的なアプローチ

    理論的なアプローチは，より長い歴史の延長線上にあり学術的で数学的なアプローチを取る．実際の現象は複雑なため太刀打ちできないので現象を理想化して解析する．そのため，実際に作ったロボットの真の最適解にはならないかもしれないが，全体的によく成り立つ効率的な魚ロボットの設計につながるだろう．

    一方，探索的なアプローチは，実際に作ったロボットに対して，試行錯誤を繰り返すことで，最適な解を見つけだす．

    自立型魚ロボットの開発¶

    これまで外部にあった装置を魚ロボット内部に搭載し，防水処理を施して，独立した動作が可能な状態にする．

    1. Raspberry PiではなくよりコンパクトなRaspberry Pi Zero Wを使用する
    2. プログラムの移植
    3. 理想的な最大動作時間とバッテリーのサイズを自分で設定し，そこから利用するバッテリー選定する
    4. 防水は，ラズパイやバッテリーをケースで覆うのではなく，液体ゴムや防水プラスチックを使用して防水処理を行おうと考えている．
      • 具体的な製品の選定が必要．
    5. 防水処理の実施
    6. システムの統合
    7. テストとデバッグ
    8. 問題点の修正

    ラズパイゼロの初期設定・ローカルPCで直接ラズパイゼロのファイルを編集する開発環境を構築する

    Raspberry Pi Imagerを使ってSDカードに書き込む．Ubuntuを簡単にインストールしきれなかったので，Rasbianをインストールした．Raspberry Pi Imagerを使ってOSを選択して，SDカードに書き込むと問題なく起動した．

    まずは，Raspberry Pi Zero 2 WをUSBでMacに接続する（特別なUSBのプラグはいらない．miniUSBとUSB-cを繋ぐケーブルがありさえすればいい）．そして，sshでリモートログイン．

    次に，ローカルPCにラズパイをマウントする．

    1. 適当な名前のディレクトリを作成（ここでは，pi_mountというディレクトリを作成）
    2. sshfsを使ってRaspberry Piの/home/piディレクトリをローカルのpi_mountディレクトリにマウントする
    3. マウントされたディレクトリに移動して，その内容を確認

    sshfsをMacでインストールしておく必要がある（とても面倒だった記憶がある）． この方法で，ローカルな環境でRaspberry Piのファイルを編集することができる．

    リモートがローカルのネットワークを利用できるようにする．

    ローカルPCで：

    ssh -D 1080 -N -f pi@pi.local

    ローカルPCで接続．接続後にプロキシを設定する．

    ssh pi@pi.local
export http_proxy="socks5://localhost:1080"
export https_proxy="socks5://localhost:1080"

    大学のプロキシを設定する必要があるかもしれない． 必要なプログラムをインストールする．

    sudo -E apt update
sudo -E apt upgrade -y

sudo -E apt install emacs
sudo -E apt install code -y

sudo -E apt install g++ -y
sudo -E apt install cmake -y
sudo -E apt install ccache -y
sudo -E apt install python3 python3-pip -y

sudo -E apt install realvnc-vnc-server -y
sudo systemctl enable vncserver-x11-serviced.service
sudo systemctl start vncserver-x11-serviced.service

sudo -E raspi-config

    Interfacing Options -> VNC -> Enableを選択．

    pi-sugarの設定¶

    設置向きに注意，ラズパイのGPIOPINにpisugarのピンがつく．

    Swapメモリの容量に要注意

    sudo apt upgradeをした際に，swapメモリの容量がいっぱいで，フリーズすることがあるので，swapメモリの状況を確認し必要であれば増やす． 

    $ sudo swapon --show
NAME      TYPE SIZE  USED PRIO
/var/swap file 100M 98.5M   -2

    この場合，swapメモリが100Mで98.5M使われている．逼迫しているので，swapメモリを増やす．

    sudo swapoff /var/swap
# 現在のswapファイルを無効にします。これにより、システムがこのファイルをswapとして使用するのを停止します。

sudo rm /var/swap
# 現在のswapファイルを削除します。このステップは、新しいswapファイルを作成する前に既存のファイルを消去するためのものです。

sudo dd if=/dev/zero of=/swapfile bs=1024 count=1048576
# 新しいswapファイルを作成します。bs=1024は1ブロックのサイズが1024バイト（1キロバイト）であることを意味し、count=1048576はそのブロックを1048576個コピーすることを意味します。結果として1GBのファイルが作成されます。

sudo chmod 600 /swapfile
# swapファイルのパーミッションを600に設定します。これは、所有者が読み書きできるが、他のユーザーはアクセスできない設定です。

sudo mkswap /swapfile
# /swapfileをswap領域として初期化します。これにより、このファイルがswapとして使用可能になります。

sudo swapon /swapfile
# 新しく作成したswapファイルを有効にします。これにより、システムがこのファイルをswap領域として使用し始めます。

echo "/swapfile swap swap defaults 0 0" | sudo tee -a /etc/fstab
# /etc/fstabファイルに新しいswapファイルのエントリを追加します。これにより、システムが再起動時にこのswapファイルを自動的に有効にします。

sudo swapon --show
# 現在有効なswap領域の情報を表示します。新しいswapファイルが正しく有効になっていることを確認できます。

sudo free -h
# メモリとswapの使用状況を表示します。新しいswap領域がシステムに追加されたことを確認できます。

    その後sudo apt upgradeでエラーが生じたので，sudo dpkg --configure -aとして，エラーを解消した．
